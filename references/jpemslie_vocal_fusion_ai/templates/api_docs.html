VocalFusion — REST API v1

Programmatic access to the VocalFusion fusion engine. Integrate from DAW plugins, bots, or external tools.  ← Back to UI

Authentication

Every request (except this page) requires a Bearer token in the Authorization header. Your first key is printed to the server's stdout on first launch. Generate additional keys via POST /api/v1/keys.

Header format
Authorization: Bearer vf_<32 hex chars>

Endpoints

POST /api/v1/fuse auth required

Submit two audio files for fusion. Returns a job_id immediately — poll /api/v1/status/<id> for progress. Max 3 concurrent jobs per key.

Request — multipart/form-data
song_a file — beat / instrumental song_b file — vocal source
Response 202
{
  "job_id":     "a1b2c3d4",
  "status_url": "/api/v1/status/a1b2c3d4",
  "share_url":  "/share/a1b2c3d4"
}
Example
curl -X POST http://localhost:5000/api/v1/fuse \
  -H "Authorization: Bearer vf_YOUR_KEY_HERE" \
  -F "song_a=@beat.mp3" \
  -F "song_b=@vocals.mp3"
GET /api/v1/status/<job_id> auth required

Poll for job progress. status is one of running / done / error.

Response 200 — running
{
  "status":      "running",
  "progress":    42,          // 0–100
  "message":     "Separating stems…",
  "api_version": "v1"
}
Response 200 — done
{
  "status":      "done",
  "progress":    100,
  "message":     "Complete",
  "output_url":  "/output/a1b2c3d4_fusion_radio.wav",
  "variants": {
    "radio":    "/output/a1b2c3d4_fusion_radio.wav",
    "club":     "/output/a1b2c3d4_fusion_club.wav",
    "intimate": "/output/a1b2c3d4_fusion_intimate.wav"
  },
  "score":       88,
  "share_url":   "/share/a1b2c3d4",
  "api_version": "v1"
}
Example
curl http://localhost:5000/api/v1/status/a1b2c3d4 \
  -H "Authorization: Bearer vf_YOUR_KEY_HERE"
GET /api/v1/keys auth required

Returns metadata for the authenticated key.

Response 200
{
  "key":         "vf_abc123...",
  "name":        "My App",
  "created_at":  "2026-03-17T00:00:00Z",
  "usage_count": 42,
  "active":      true
}
Example
curl http://localhost:5000/api/v1/keys \
  -H "Authorization: Bearer vf_YOUR_KEY_HERE"
POST /api/v1/keys auth required

Generate a new API key. Requires an existing valid key. Store the returned key immediately — it won't be shown again through this endpoint.

Request body — application/json
{ "name": "My DAW Plugin" }
Response 201
{
  "key":        "vf_newkey...",
  "name":       "My DAW Plugin",
  "created_at": "2026-03-17T12:00:00Z"
}
Example
curl -X POST http://localhost:5000/api/v1/keys \
  -H "Authorization: Bearer vf_YOUR_KEY_HERE" \
  -H "Content-Type: application/json" \
  -d '{"name": "My DAW Plugin"}'
DELETE /api/v1/keys/<key> auth required

Revoke an API key. You can only revoke your own key — use the full key value in the URL path.

Response 200
{ "revoked": true, "key": "vf_abc123..." }
Example
curl -X DELETE http://localhost:5000/api/v1/keys/vf_YOUR_KEY_HERE \
  -H "Authorization: Bearer vf_YOUR_KEY_HERE"

Error Format

All errors return a consistent JSON envelope with a human-readable message and a machine-readable code. 

{ "error": "human readable message", "code": "ERROR_CODE" }
Code HTTP Status Meaning
UNAUTHORIZED 401 Missing, invalid, or revoked API key
NOT_FOUND 404 Unknown job_id
RATE_LIMITED 429 More than 3 concurrent jobs for this key
BAD_REQUEST 400 Missing files or malformed request body

Getting started

Your default API key is printed to the server's stdout the first time VocalFusion starts. It looks like: vf_<32 hex chars>. Use it to authenticate all requests and generate additional keys as needed. 

# 1. Check your key metadata
curl http://localhost:5000/api/v1/keys \
  -H "Authorization: Bearer vf_YOUR_KEY"

# 2. Start a fusion job
JOB=$(curl -s -X POST http://localhost:5000/api/v1/fuse \
  -H "Authorization: Bearer vf_YOUR_KEY" \
  -F "song_a=@beat.mp3" \
  -F "song_b=@vocals.mp3" | jq -r .job_id)

# 3. Poll until done
while true; do
  STATUS=$(curl -s http://localhost:5000/api/v1/status/$JOB \
    -H "Authorization: Bearer vf_YOUR_KEY" | jq -r .status)
  echo $STATUS
  [ "$STATUS" != "running" ] && break
  sleep 5
done
