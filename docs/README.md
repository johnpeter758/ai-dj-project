# Docs index

This repo keeps durable docs in a few buckets so GitHub stays readable.

## Core docs
- `../README.md` — main repo overview and runnable entry points
- `notes/TARGET_ARCHITECTURE.md` — intended long-term system shape
- `notes/CORE_LAYOUT.md` — current code layout
- `notes/WORK_LOG.md` — running engineering notes

## Render docs
Render-first design notes and deterministic renderer contracts live in `render/`.

Suggested reading order:
1. `render/RENDER_V1_CONTRACT.md`
2. `render/RENDER_RESOLVER_DESIGN_2026-03-11.md`
3. `render/RENDER_SYNTHETIC_TEST_SUITE_2026-03-11.md`
4. `render/RENDER_OVERLAP_OWNERSHIP_POLICY_2026-03-11.md`

The remaining files in `render/` are checkpoint notes from concrete prototype/debug passes.

## Research docs
Research summaries and external pattern reviews live in `research/`.

## Notes
Operational architecture notes, classification/cleanup notes, and work log live in `notes/`.
